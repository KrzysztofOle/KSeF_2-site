What the project does

Three boundaries. One controlled flow.

KSeF_2 separates public capabilities, KSeF protocol details and ERP-specific behavior so the core remains testable and technology-neutral.

01 / COMMUNICATE

KSeF API

Certificate authentication, typed outcomes, bounded transport retries and short-lived sessions behind a dedicated Gateway.

02 / PRESERVE

Native document

Original XML remains the source document. Company isolation, correlation and integrity checks are explicit.

03 / ADAPT

Comarch Optima

Optima SQL structures and projections stay inside a Provider informed by controlled snapshots, traces and discoveries.

Current status

Honest gates, visible limits.

Implemented means proven in the declared scope. Research findings never silently become permission for direct SQL writes.

Implemented

  • typed and redacted result
  • no invoice or Optima operation

Evidence

  • durable Discovery registry

Architecture

Ports at every volatile edge.

The core coordinates business capabilities. Providers implement explicit ports; adapters translate representations and transport concerns.

Public APITransport-neutral requests, authorization and typed outcomes.
→
Application CoreCompany Context, workflows, coordinators and native documents.
→
ProvidersKSeF Gateway, configuration, storage and Optima-specific integration.
Company is the execution and isolation boundary.
Core never interprets ERP-specific storage as domain truth.
Credentials and sessions never cross public boundaries.
Native XML is preserved before projections are created.
Safety gate: current evidence does not authorize direct writes to a production Optima database.
